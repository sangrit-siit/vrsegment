
# OVERVIEW : vrsegment

vrsegment is a python library for Thai word segmentation.
This library consists of 

    - segmentor.py as the segmentor itself
    - src/test and src/key as test and key file
    - src/corpus as the dictionary of thai words, the method of gathering will be discussed later

# PART 2. Using vrsegment library

There are 2 algorithms implemented in this library.
    1. forwardCut
    2. backwardCut
    
Both use the idea of longest matching algorithm but with different implementation, they also are dictionary based algorithm. The dictionary(corpus) we are using will be discussed later. 

Let's first begin with the first algorithm, frontcut.

## 2.1 forwardCut

First, to import the library we use



```python
import vrsegment as vr
```

Let's try with some test sentence 


```python
test_sentence1 = "ฉันนั่งตกปลาอยู่ริมตลิ่ง"
answer = vr.forwardCut(test_sentence1)
print("|".join(answer))
```

    ฉัน|นั่ง|ตก|ปลา|อยู่|ริม|ตลิ่ง



```python
test_sentence2 = "แปลกใจฉันจริงปลาไม่กินเหยื่อ"
answer = vr.forwardCut(test_sentence2)
print("|".join(answer))
```

    แปลกใจ|ฉัน|จริง|ปลา|ไม่|กิน|เหยื่อ


This two sentences are simple and could be handled very well.

But in practical use, there are more complex sentences and this forwardCut algorithm might not work properly.
Such as some situation below.
    - Sentences containing 'ๆ'


```python
complex_sentence1 = "เรื่อยๆมาเรียงๆนกบินเฉียงไปทั้งหมู่"
answer = vr.forwardCut(complex_sentence1)
print("|".join(answer))
```

    เรื่อยๆ|มา|เรียง|ๆ|นก|บิน|เฉียง|ไป|ทั้ง|หมู่


    - Sentences containing numbers, symbol or something that isn't thai alphabets


```python
complex_sentence2 = "1 2 3 Ok! Go... ไปไป"
answer = vr.forwardCut(complex_sentence2)
print("|".join(answer))
```

    1 2 3 Ok! Go... |ไป|ไป


Though the algorithm doesn't work properly when handle non-thai alphabets. But when the algorithm handle unknown thai words or words that are not in the corpus, the algorithm still work well because we have implemented a loop that will skip some letters to look for more known words.

Try to see this case


```python
unknown_sentence1 = "สาวบึงกาฬบอกรัก ก็แปลว่าฮักหลายหลาย"
answer = vr.forwardCut(unknown_sentence1)
print("|".join(answer))
```

    สาว|บึง|กา|ฬ|บอก|รัก| |ก็|แปล|ว่า|ฮัก|หลาย|หลาย


The words 'บึงกาฬ', 'ฮัก' are not in the corpus. But the algorithm will skip character by character to search for known words which results in segmenting an unknown words together as one word.

There are also more test cases and will be in PART 3 in which we compare two algorithms with our 100 test sentences collated from several sources such as books, newspaper or even some articles from social media.

## 2.2 backwardCut

To use backwardCut we need to


```python
import vrsegment as vr
```

Try with those sentences above.


```python
test_sentence1 = "ฉันนั่งตกปลาอยู่ริมตลิ่ง"
answer = vr.backwardCut([test_sentence1,'',''])
"|".join(answer)
```




    'ฉัน|นั่ง|ตก|ปลา|อยู่|ริม|ตลิ่ง'




```python
test_sentence2 = "แปลกใจฉันจริงปลาไม่กินเหยื่อ"
answer = vr.backwardCut([test_sentence2,'',''])
"|".join(answer)
```




    'แปลกใจ|ฉัน|จริง|ปลา|ไม่|กิน|เหยื่อ'



### Remark
* using backwardCut will need to provide special list object to the function which is
  ```  
    sentence_obj = [input_sentence,'',''] 
    answer = vr.backwardCut(sentence_obj)
   ``` 
* and printing the answer must use recursive printing function with following parameters
    ```
    vr.recursivePrint(answer, "|")
    ```
* both algorithm uses the same corpus.


```python
complex_sentence1 = "เรื่อยๆมาเรียงๆนกบินเฉียงไปทั้งหมู่"
answer = vr.backwardCut([complex_sentence1,'',''])
"|".join(answer)
```




    'เรื่อยๆ|มา|เรียง|ๆ|นก|บิน|เฉียง|ไป|ทั้ง|หมู่'




```python
complex_sentence2 = "1 2 3 Ok! Go... ไปไป"
answer = vr.backwardCut([complex_sentence2,'',''])
"|".join(answer)
```




    '1| |2| |3| |O|k|!| |G|o|.|.|.| |ไป|ไป'




```python
unknown_sentence1 = "สาวบึงกาฬบอกรัก ก็แปลว่าฮักหลายหลาย"
answer = vr.backwardCut([unknown_sentence1,'',''])
"|".join(answer)
```




    'สาว|บึง|กา|ฬ|บอก|รัก| |ก็|แปล|ว่า|ฮัก|หลาย|หลาย'

# PART 4. Analysis on the errors

There are a few points we noticed as difficulties of Thai Words Segmentation. We're discussing about those points about the problem and our solutions we come up with.

1. Lacking of the word in the corpus
    - Such as lacking of specific noun 'วิสามาณญนาม'
    - We try to overcome this problem by expanding our corpus and creating skip method in order to search for more known words.
    
2. Ambiguous definition of Thai words type
    - This causes unconsistency in 'the correct segmentation'. 
    - See the example of 'ซ่อมบำรุง' 
        - This could be segmented as ซ่อม|บำรุง as both words combined do not fall in the category of 'คำประสม'
        - But longdo dictionary has the word ซ่อมบำรุง as one whole word.
    - Another error is 'อาการนาม' which is the word that combine 'การ'/'ความ' with 'คำวิเศษณ์'
        - Almost half of the error in the test set came from not having all the 'อาการนาม' in the corpus.
    - We try to overcome this problem with longest matching algorithm.
    
3. Segmentation that requires context
    - See this example
    ```
    >> input:	เนื่องจากว่าถุงมือฟุตบอลเป็นสิ่งที่ใช้งานหนักและเป็นแหล่งสะสมของคราบสกปรกแบคทีเรียและคราบเหงื่ออยู่เสมอ
    >> key:		เนื่องจาก|ว่า|ถุงมือ|ฟุตบอล|เป็น|สิ่ง|ที่|ใช้|งาน|หนัก|และ|เป็น|แหล่ง|สะสม|ของ|คราบ|สกปรก|แบคทีเรีย|และ|คราบ|เหงื่อ|อยู่|เสมอ
    >> output 1:	เนื่อง|จาก|ว่า|ถุงมือ|ฟุตบอล|เป็น|สิ่ง|ที่|ใช้|งาน|หนัก|และ|เป็น|แหล่ง|สะสม|ของ|คราบ|สกปรก|แบคทีเรีย|และ|คราบ|เหงื่อ|อยู่|เสมอ
    >> output 2:	เนื่อง|จา|กว่า|ถุงมือ|ฟุตบอล|เป็น|สิ่ง|ที่|ใช้|งาน|หนัก|และ|เป็น|แหล่ง|สะสม|ของ|คราบ|สกปรก|แบคทีเรีย|และ|คราบ|เหงื่อ|อยู่|เสมอ
    ```
        - We could see that 'จากว่า' becomes 'จาก|ว่า' in output1 and 'จา|กว่า' in output2. To cut this kind of words, cutting from the from and cutting from the back will not create the correct result always.
    - We really couldn't do anything about it unless there is some algorithm that can learn the context.
    - We have one idea which isnt't implemented which is creating a network of thai words - 'Thai Words Graph'.
        - A network means a directed graph of node as word and path as existing connection of two words
        - Traversing the graph to see the connection might help the algorithm see the context of the words to be cut.
        - This could be viewed as a Corpus with relations of words.

Using [ORCHID Thai POS Tagging](http://www.academia.edu/1215347/ORCHID_Thai_part-of-speech_tagged_corpus) Table 5 : Thai POS Tagset as the standard of Thai language seems a very feasible option.
